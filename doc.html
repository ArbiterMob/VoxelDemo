Progetto Modulo 2 - Fondamenti di Computer Graphics

VoxelDemo

1. Descrizione dell'applicazione

Il progetto è una demo interattiva sviluppata con la libreria Three.js, fortemente ispirata alle meccaniche del videogioco voxel Minecraft. Pur non avendo un vero e proprio obiettivo, l'applicazione funge da vetrina tecnica per esplorare e testare cosa la libreria offre. Viene generato staticamente un mondo di blocchi (voxelWorld) esplorabile e interattivo.

In particolare, la demo mostra l'implementazione e la gestione dei successivi componenti fondamentali:

  • Illuminazione e Ombre: Gestione dinamica tramite l'uso di DirectionalLight e AmbientLight.
  • Nebbia: Implementazione della nebbia volumetrica (Fog) per dare profondità alla scena.
  • Gestione Camere: Sistemi di visuale a due camere intercambiabili (PerspectiveCamera).
  • Interazione 3D: Proiezione di raggi nello spazio tramite Raycaster per l'interazione con i blocchi.
  • Controlli Navigazione: Supporto per controlli differenti tramite PointerLockControls e OrbitControls.
  • Ottimizzazione: Generazione procedurale e dinamica delle geometrie tramite BufferGeometry per alleggerire il rendering.
  • Modellazione Esterna: Importazione di un modello 3D attraverso il modulo GLTFLoader.
  • Interfaccia Utente: UI gestita tramite la libreria lil-gui.
  • Post-Processing: Tecniche di post-processing per arricchire la scena.

2. Spiegazione delle scelte effettuate

Ottimizzazione delle Geometrie (VoxelWorld)

Per massimizzare l'efficienza dell'applicazione, si è scelto di non instanziare una BoxGeometry separata per ciascun voxel. Questo approccio avrebbe saturato rapidamente la memoria. Al contrario, l'applicazione definisce una classe personalizzata VoxelWorld che raggruppa i voxel in chunk chiamati "celle" (con dimensione 32x32x32). La geometria viene costruita dinamicamente: l'algoritmo calcola i vertici, le normali e le coordinate UV aggiungendo unicamente le facce visibili del blocco. Verificando la presenza di blocchi adiacenti, la faccia condivisa non viene generata, riducendo drasticamente il numero di triangoli da renderizzare.

generateGeometryDataForCell(cellX, cellY, cellZ) {
    const {cellSize, tileSize, tileTextureWidth, tileTextureHeight} = this;

    const position = [];
    const normals = [];
    const uvs = [];
    const indices = [];

    const startX = cellX * cellSize;
    const startY = cellY * cellSize;
    const startZ = cellZ * cellSize;

    for (let y = 0; y < cellSize; y++) {
        const voxelY = startY + y;
        for (let z = 0; z < cellSize; z++) {
            const voxelZ = startZ + z;
            for (let x = 0; x < cellSize; x++) {
                const voxelX = startX + x;
                const voxel = this.getVoxel(voxelX, voxelY, voxelZ);
                if (voxel) {
                    const uvVoxel = voxel - 1; // voxel 0 is sky so for UVs we start at 0
                    for (const {dir, corners, uvRow} of VoxelWorld.faces) {
                        
                        const neighbour = this.getVoxel(
                            voxelX + dir[0],
                            voxelY + dir[1],
                            voxelZ + dir[2]);
                        
                        if (!neighbour) {
                            const ndx = positions.length / 3;
                            for (const {pos, uv} of corners) {
                                positions.push(pos[0] + voxelX, pos[1] + voxelY, pos[2] + voxelZ);
                                normals.push(...dir);
                                uvs.push(
                                    (uvVoxel + uv[0]) * tileSize / tileTextureWidth,
                                    1 - (uvRow + 1 - uv[1]) * tileSize / tileTextureHeight
                                );
                            }
                            indices.push(ndx, ndx + 1, ndx + 2,
                                ndx + 2, ndx + 1, ndx + 3);
                        }
                    }
                }
            }
        }
    }

    return {positions, normals, uvs, indices};
}

Gestione delle Collisioni (Collision Detection)

Il movimento in prima persona prevede due differenti strategie per la rilevazione delle collisioni, alternabili per dimostrare le differenze architetturali e di performance:

  • Metodo Raycaster: Sfrutta istanze multiple di THREE.Raycaster per calcolare le intersezioni in sei direzioni differenti rispetto al giocatore. Sebbene intuitivo, questo metodo comporta un costo computazionale elevato che può causare notevoli cali di frame rate (da 300 fps a 60 fps con tanti blocchi intorno).
  • Metodo Axis-Aligned Bounding Box (AABB) Grid Sweep : Estremamente più efficiente e accurato, questo approccio calcola preventivamente la posizione futura in cui il giocatore intende muoversi. Utilizza poi la funzione getVoxel(x, y, z) per verificare direttamente nell'array dei dati se lo spazio è occupato. Eseguendo il controllo sia ai piedi che all'altezza degli occhi, garantisce collision detection senza alcun calo di prestazioni.
const checkCollision = (px, py, pz, yOffset, yHeight) => {
    const minX = Math.floor(px - playerRadius);
    const maxX = Math.floor(px + playerRadius);
    const minZ = Math.floor(pz - playerRadius);
    const maxZ = Math.floor(pz + playerRadius);
    const minY = Math.floor(py + yOffset);
    const maxY = Math.floor(py + yHeight);
    
    for (let y = minY; y <= maxY; y++) {
        for (let z = minZ; z <= maxZ; z++) {
            for (let x = minX; x <= maxX; x++)  {
                if (world.getVoxel(x, y, z) > 0) 
                    return true;
            }
        }
    }

    return false;
};

L'Avatar del Giocatore (Mesh Davy Jones)

Per rappresentare il corpo del giocatore è stata integrata la mesh del personaggio "Davy Jones" (scansionata in classe). Tramite l'uso di GLTFLoader, il modello viene caricato e assegnato a un MeshStandardMaterial. Per ovviare ai problemi di clipping visivo e non ostruire la vista, il modello viene reso invisibile durante la navigazione in prima persona. Passando alla telecamera libera (Free Camera), la mesh riappare e viene traslata esattamente nel punto in cui risiedeva la telecamera, simulando così l'osservazione del proprio avatar dall'esterno.

3. Funzionalità previste ed interattività

Sistemi di Telecamera

L'utente ha a disposizione due distinti Punti di Vista (POV):

  • Free Camera: Permette di esplorare la scena liberamente ruotando e zoomando attorno a un punto focale tramite la classe OrbitControls.
  • First Person Camera: Offre un'esperienza immersiva. Il cursore viene bloccato nello schermo (tramite PointerLockControls) e compare un mirino centrale. Il movimento avviene sfruttando i classici comandi direzionali (W, A, S, D), mentre la barra spaziatrice permette di saltare.

Posizionamento e Rimozione dei Voxel

L'interazione con i blocchi avviene tramite la proiezione di un raggio dal centro della telecamera. Il raggio possiede una distanza d'azione massima (8 unità in prima persona, 200 in telecamera libera) per limitare le modifiche a zone realisticamente raggiungibili.

  • La selezione del blocco da posizionare avviene cliccando sull'interfaccia ad icone in alto a sinistra, oppure scorrendo la rotella del mouse quando ci si trova in prima persona.
  • Per piazzare un blocco è sufficiente cliccare il tasto sinistro del mouse dopo aver selezionato una tipologia valida.
  • Per distruggere un blocco, l'utente può cliccare il tasto sinistro senza alcun blocco selezionato nella UI, oppure tenere premuto il tasto Shift durante il click sinistro (ignorando il blocco selezionato).
intersectRay(normalizedPosition, scene, camera, maxDistance) {
    const raycaster = new THREE.Raycaster();
    raycaster.setFromCamera(normalizedPosition, camera);
    raycaster.far = maxDistance;

    const intersectedObjects = raycaster.intersectObjects(scene.children);
    const intersection = intersectedObjects.find(intersect => intersect.object instanceof THREE.Mesh && intersect.face);

    if (intersection) {
        const point = intersection.point;
        const normal = intersection.face.normal;

        const voxelX = Math.floor(point.x - normal.x * 0.5);
        const voxelY = Math.floor(point.y - normal.y * 0.5);
        const voxelZ = Math.floor(point.z - normal.z * 0.5);
        const voxel = this.getVoxel(voxelX, voxelY, voxelZ);

        return {position: point, normal: normal, voxel};
    }
}

function placeVoxel(event) {
    let x, y;
    let maxDistance = 200;
    
    if (activeCamera === fpCamera) {
        x = 0;
        y = 0;
        maxDistance = 8;
    }
    else 
    {
        const pos = getCanvasRelativePosition(event);
        x = (pos.x / canvas.width ) *  2 - 1;
        y = (pos.y / canvas.height) * -2 + 1;
    }

    const intersection = world.intersectRay({x, y}, scene, activeCamera, maxDistance);
    if (intersection) {
        const voxelId = event.shiftKey ? 0 : currentVoxel;

        // the intersection point is on the face. That means
        // the math imprecision could put us on either side of the face.
        // so go half a normal into the voxel if removing (currentVoxel = 0)
        // our out of the voxel if adding (currentVoxel  > 0)
        const offset = voxelId > 0 ? 0.5 : -0.5;
        const voxelX = Math.floor(intersection.position.x + intersection.normal.x * offset);
        const voxelY = Math.floor(intersection.position.y + intersection.normal.y * offset);
        const voxelZ = Math.floor(intersection.position.z + intersection.normal.z * offset);
            
        world.setVoxel(voxelX, voxelY, voxelZ, voxelId);
        updateVoxelGeometry(voxelX, voxelY, voxelZ);
    }
}

4. Particolarità tecniche

Effetti di Post-Processing

Oltre al normale flusso di rendering, è stato implementato un sistema avanzato sfruttando EffectComposer. L'utente può abilitare diversi shader dinamicamente dal pannello della GUI:

Effetto Descrizione
Bloom + Film Aggiunge un alone luminoso brillante ai punti di luce (BloomPass), accoppiato a grana da pellicola cinematografica e leggera distorsione ottica (FilmPass).
DotScreen + RGBShift Applica un retino grafico per lo schermo affiancato allo sfalsamento dei canali cromatici, simulando un effetto di aberrazione cromatica.
Halftone Simula la tecnica del retino tipografico.
Pixelated Renderizza la scena a bassa risoluzione aggiungendo linee di profondità, creando un effetto "Pixel-Art". 
// Esempio del filtro Bloom+Film
const composer = new EffectComposer(renderer)
    
const renderPass = new RenderPass(scene, camera);
composer.addPass(renderPass);

const bloomPass = new BloomPass(1, 25, 4, 256);
composer.addPass(bloomPass);
const filmPass = new FilmPass(0.5, false);
composer.addPass(filmPass);

Interfaccia GUI e Architettura Modulare

Tutta la scena e i suoi parametri sono gestiti dinamicamente tramite lil-gui. Per interfacciarsi correttamente agli oggetti complessi di Three.js, il file utils.js espone delle classi "Helper" basate sulle linee guida ufficiali. Queste classi (come ColorGUIHelper e MinMaxGUIHelper) fungono da ponte, trasformando vettori e colori nativi in stringhe e valori lineari gestibili dagli slider.

Texturing tramite Atlas

Il texturing del mondo non avviene caricando un'immagine per ogni singolo materiale, ma attraverso un approccio ottimizzato chiamato Texture Atlas (atlasPOT.png). Si tratta di un'unica immagine che racchiude i design di tutti i blocchi, dimensionata in potenza di due per ottimizzare il mip mapping. La stessa immagine viene riutilizzata nell'HTML per generare le icone della UI, modificando il posizionamento tramite attributi CSS.
